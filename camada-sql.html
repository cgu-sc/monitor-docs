Camada SQL

Views e stored procedure — numerados pela ordem de dependência

⚙ Técnico

Todos os objetos residem no schema temp_CGUSC.compras. A hierarquia de arquivos segue a ordem de dependência: views por fonte → views consolidadas → stored procedure.

vw_disp_inex_pncp_sc

vw_licitacoes_consolidada_sc
06_-_vw_licitacoes_consolidada_sc.sql · View principal
CONSOLIDADA

UNION ALL das quatro fontes com anti-joins. Cada licitação aparece exatamente uma vez, pela fonte de maior prioridade.

SQL — UNION ALL com Anti-joins
SELECT * FROM vw_licitacoes_faseexterna_sc
UNION ALL
SELECT * FROM vw_licitacoes_comprasnet_sc c
WHERE NOT EXISTS (SELECT 1 FROM vw_licitacoes_faseexterna_sc f WHERE f.Cod_licitacao = c.Cod_licitacao)
UNION ALL
SELECT * FROM vw_licitacoes_siasg_sc s
WHERE NOT EXISTS (...faseexterna...) AND NOT EXISTS (...comprasnet...)
UNION ALL
SELECT * FROM vw_disp_inex_pncp_sc p
WHERE NOT EXISTS (...3 fontes anteriores...)

Views de Itens

vw_itens_faseexterna_sc
07_-_vw_itens_faseexterna_sc.sql
FaseExterna1ª (Itens)

Itens com fornecedor vencedor (via proposta_item e participacao), valores calculados e classificação CATMAT/CATSER via JOIN com Siasg DW.

SQL — Hierarquia de 4 valores adjudicados (v2.0.9)
-- Condição: item ativo (situacao='1') E proposta adjudicada (situacao='6')
-- item.homologado removido (v2.0.9) — campo não confiável, 35% falsos negativos
CASE WHEN i.situacao = '1' AND pi.situacao = '6' THEN
    COALESCE(qtde_adjudicada, quantidade_ofertada, quantidade_solicitada, 1)
    * COALESCE(
        valor_negociado_julgamento_calculado,   -- 1º: pós-negociação
        valor_lance_calculado,                  -- 2º: pós-disputa
        melhor_valor_proposta_lance_calculado,  -- 3º: melhor valor
        valor_proposta_calculado                -- 4º: proposta inicial (Concorrências fechadas)
    )
ELSE NULL END AS valor_adjudicado_calculado
vw_itens_siasg_sc
10_-_vw_itens_siasg_sc.sql
Siasg DW2ª (Itens)

Maior riqueza de dados: CATMAT/CATSER completo, dimensões do fornecedor (UF, município, porte, natureza jurídica), quantidade de lances e valores homologados.

⚠️
JOIN de classificação via tabela fato (correção v2.0.7): usar f_item.ID_ITCP_TP_COD_MAT_SERV, não d_item.ID_ITCP_MATERIAL_SERVICO — a versão anterior produzia resultados incorretos.
vw_itens_comprasnet_sc
08_-_vw_itens_comprasnet_sc.sql
Comprasnet4ª (Itens)

UNION ALL das bases sigilosa e não-sigilosa. Campos ausentes preenchidos com CAST(NULL AS tipo) para compatibilidade no UNION ALL — NULL sem CAST causa erro de tipo.

vw_itens_consolidada_sc
09_-_vw_itens_consolidada_sc.sql · View principal de itens
CONSOLIDADA

Entrega ao Power BI

Stored Procedure

sp_itens_licitacoes_pbi
sp_itens_licitacoes_pbi.sql · Stored Procedure para Power BI
SP

Resolve +1h → ~5–8 min via 3 tabelas temporárias (#temp_censura, #temp_cnpjs_itens, #temp_cnpj). Aplica censura de sigilo em 10 campos monetários e enriquece o fornecedor com dados da Receita Federal (db_cnpj): natureza jurídica, tipo e porte com descrição legível.

SQL — Chamada no Power Query (modo Importação)
EXEC temp_CGUSC.compras.sp_itens_licitacoes_pbi

10 campos monetários censurados

valor_estimado valor_estimado_calculado valor_unitario_homologado melhor_valor_proposta_lance_calculado melhor_valor_proposta_lance_informado melhor_valor_nao_desclassificado_calculado valor_proposta_calculado valor_lance_calculado valor_negociado_julgamento_calculado valor_adjudicado_calculado
ℹ️
Não censurados: qtd_lances_item e qtd_fornecedores_propostas — não revelam o orçamento sigiloso.
ABS() aplicado (v2.0.11): numero_grupo, quantidade_solicitada e quantidade_minima_solicitada para tratar valores negativos de cadastro.

Cabeçalhos de grupo filtrados na SP

A SP entrega apenas subitens ao Power BI. Cabeçalhos (numero_item < 0) são excluídos — já serviram para calcular o valor adjudicado na view de licitações, e exibi-los causaria duplicação no painel.

SQL — Filtro na SP
-- Exclui cabeçalhos de grupo; mantém subitens e itens avulsos
WHERE i.numero_item > 0 OR i.numero_item IS NULL
