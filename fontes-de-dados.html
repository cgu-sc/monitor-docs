Fontes de Dados

Os quatro sistemas federais + fonte auxiliar que alimentam o Monitor de Compras

Visão Geral

As Cinco Fontes

Cada fonte tem cobertura temporal e características distintas. A hierarquia garante que, quando uma licitação existe em múltiplas fontes, sempre prevalece a mais completa. O Alice é uma fonte auxiliar — não contém licitações, mas fornece dados essenciais que as outras fontes não têm.

FaseExterna
1ª Prioridade

Novo portal ComprasGov. Cobre todas as modalidades de 2024+, exceto Inexigibilidade (modal. 7). Fonte mais completa e atualizada para licitações recentes.

Portal legado. Pregão e RDC. Valores adjudicados obtidos via Siasg DW — a tabela TBL_PROPOSTAITEM não está disponível na CGU.

1ª Prioridade — Licitações e Itens

ComprasGov FaseExterna

Sistema atual do SERPRO. Cobre todas as modalidades exceto Inexigibilidade (modal. 7).

TabelaConteúdo
compraLicitações — dados gerais, situação, modalidade, valores, datas
itemItens — estimativas, código CATMAT/CATSER, sigilo
proposta_itemPropostas dos fornecedores — lances, valores negociados
participacaoParticipação dos fornecedores — identificação do vencedor
⚠️
6 registros idênticos por licitação: A tabela compra não guarda histórico de versões. Usar sempre ROW_NUMBER() para deduplicar.
⚠️
Itens recriados com novos IDs (mesmo versao=0): Alterações significativas geram novos IDs com o mesmo numero_item. Sempre usar ORDER BY versao DESC, id DESC — correção aplicada em v2.0.9.
ℹ️
Sigilo centralizado: Valores sigilosos ficam no mesmo banco (db_compras). O campo orcamento_sigiloso = 'S' apenas indica o sigilo — o valor está preenchido normalmente. Verificar periodicamente se a CGPLA mantém esta regra.
SQL — Deduplicação FaseExterna (padrão obrigatório) 
ROW_NUMBER() OVER (
    PARTITION BY i.numero_uasg, i.codigo_modalidade,
                 i.numero_compra, i.ano_compra, i.numero_grupo, i.numero_item
    ORDER BY i.versao DESC, i.id DESC  -- id DESC é o desempate crítico (v2.0.9)
) AS rn

2ª Prior. Licitações · 4ª Prior. Itens

Comprasnet

Portal legado. Na CGU, disponível em duas bases separadas por sigilo.

SQL — União das bases sigilosa e não-sigilosa 
-- Itens com valor preenchido (não sigilosos ou já homologados)
SELECT * FROM db_compras.comprasnet.tb_compras_mat
WHERE ISNULL(ipgValorRef, 0) <> 0

UNION ALL

-- Itens sigilosos com valores reais
SELECT * FROM db_compras_sigiloso.comprasnet.tb_compras_mat
⚠️
TBL_PROPOSTAITEM indisponível na CGU: A tabela com fornecedores adjudicados e valores por item não existe em db_compras. Por isso o Siasg DW é 2ª prioridade para itens — ele tem esses dados para licitações até abril/2024.

PNCP

Única fonte atualizada para Dispensas e Inexigibilidades recentes. Modalidades 8 e 9 no PNCP são normalizadas para os códigos 6 e 7 usados no restante do sistema.

FiltroCondição
Modalidademodalidadeid IN (8, 9) → normalizado para 6 e 7
Apenas ativosexcluido = 'f'
UASGs de SCJOIN com vw_jurisdicionadas_sc via unidadeorgao.codigounidade
PeríodoYEAR(datapublicacaopncp) >= 2022
⚠️
Campo numerocompra despadronizado: Pode conter valores como "060301/2024" ou "DL 1". Sem impacto prático — o PNCP só é usado para registros inexistentes nas fontes anteriores, sem necessidade de JOIN por esse campo.
⚙ Técnico
5ª Fonte — Auxiliar

Alice e e-CGU

O Alice não é fonte de licitações — fornece datas de publicação confiáveis (fonte preferencial) e alertas de irregularidades.

DadoUso
data_primeira_publicacaoData da versão 1 no Alice — preferencial em todas as fontes
data_ultima_publicacaoData da versão mais recente
Alertas AliceVinculados via licitacao_id ou pregao_id → licitacao_id
Integração e-CGUIndica se o alerta gerou tarefa no sistema e-Aud
ℹ️
Fallback de datas: Quando a licitação não existe no Alice, usa a data de inclusão da fonte. O campo data_publicacao_fonte indica a origem usada: 'ALICE', 'FASEEXTERNA', 'COMPRASNET' ou 'SIASG'.
